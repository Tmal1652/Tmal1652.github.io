ShowZ

ShowZ is a native SwiftUI iOS app for TV discovery and discussion. This page highlights product progression from V1 foundations to the current v2.8 Alpha build.

What this showcase covers

  • Version-by-version product evolution from V1 to V3.
  • Discovery UX, recommendation behavior, and chat experience changes.
  • Technical decisions around reliability, caching, and local-first architecture.

At a Glance

  • Current Stagev2.8 Alpha
  • Core PillarsDiscovery, Personalization, Room Chat
  • Primary DataTVMaze API + deterministic cache fallback
  • FocusReadability, stability, and beta prep

Current Version

v2.8 Alpha is in active development with local-first chat stabilization, readability polish, and performance tuning before beta.

Highlights

  • TVMaze-first discovery with fast search and personalization from explicit likes.
  • Distinct For You, Trending, and Popular rails for structured browsing.
  • Remote-first show detail enrichment for seasons, episodes, and summaries.
  • Immersive room-based chat with deterministic local persistence.

Latest Updates

  • Introduced local-first chat timelines with General, Season, and Episode rooms.
  • Added persisted reactions, spoiler-state handling, and optional Spoiler Guard.
  • Expanded themed chat visuals with resolver/fallback guardrails.
  • Continued reliability and performance fixes across search, detail, and chat flows.

Technical Project Notes

Implementation details for evaluating architecture decisions, data strategy, and local run setup.

Architecture

  • SwiftUI + async/await + URLSession.
  • TVMaze-first feed/search routing.
  • Detail loader/enrichment boundaries before render.

Data Strategy

  • Primary source: TVMaze API.
  • Fallback chain: memory -> 24h disk -> local catalog.
  • Personalization from explicit likes only.

Run Locally

  1. Open `ShowZ.xcodeproj`.
  2. Select `ShowZ` scheme.
  3. Choose compatible iPhone runtime.
  4. Build and run (target: iOS 26.2).

Roadmap

  • Stabilize chat behavior and history performance.
  • Expand immersive theme coverage consistency.
  • Finalize spoiler UX and evaluate backend sync post-beta.
